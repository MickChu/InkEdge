# Phase 3 — Writer Agent（章节写手）

> 状态：✅ 已完成（2026-05-23）
> 测试：18/18 通过

---

## 一、设计目标

Writer 是小说的"生产线"。Architect 建好了设定，Writer 负责把设定变成一章一章的正文。核心挑战是——如何在写第 N 章时，让模型知道前 N-1 章都发生了什么，而不是"失忆"地每一章都像全新开篇。

---

## 二、三大组件

```
WriterAgent.run(context)
  │
  ├─→ ContextBuilder (3a)   组装上下文包（≤8000字）
  │     ├─ 读取 foundation 文件
  │     ├─ 读取最近5章摘要
  │     ├─ 可选：语义检索补充
  │     └─ 可选：风格指南
  │
  ├─→ WriterAgent (3b)      生成章节正文
  │     ├─ 10条系统写作规则
  │     ├─ 内嵌章前规划（100-200字 memo）
  │     └─ 输出纯正文
  │
  └─→ StateManager (3c)     写后状态更新
        ├─ 保存章节文件
        ├─ 生成摘要追加到 chapter_summaries.md
        └─ 更新写状态
```

## 三、ContextBuilder (3a) — 上下文组装

### 3.1 问题

LLM 上下文窗口有限。把整本书塞进去不现实。需要聪明的"压缩"策略。

### 3.2 方案：混合上下文

```
┌─ 结构上下文（必须，约5500字） ─────────────┐
│ story_frame      2000字    核心冲突始终在   │
│ volume_map       1500字    当前卷的方向     │
│ roles            3000字    相关角色卡       │
│ book_rules        500字    YAML 硬限制      │
│ pending_hooks    1000字    活跃伏笔         │
│ chapter_summaries 2000字   最近5章摘要       │
│ current_state      500字   当前角色状态     │
│ 合计预算: 8000字                             │
└─────────────────────────────────────────────┘
          +
┌─ 可选上下文 ────────────────────────────────┐
│ style_guide      2000字   风格指南          │
│ semantic_hits    1500字   语义检索结果      │
└─────────────────────────────────────────────┘
```

### 3.3 优先级裁剪

当上下文超预算时（虽然当前分配合计不超过 8000），按优先级裁剪：

```
优先级: story_frame > roles > volume_map > hooks > summaries > state
```

## 四、WriterAgent (3b) — 写作逻辑

### 4.1 10条系统写作规则

这 10 条规则编码在 `WRITER_SYSTEM_PROMPT` 中，每条都对应一个常见的小说写作问题：

| # | 规则 | 解决问题 |
|---|------|----------|
| 1 | 简体中文，句子长短交替，3-5行/段 | 手机阅读体验 |
| 2 | 严格执行目标字数 | 章长失控 |
| 3 | 伏笔前后呼应 | 埋坑不填 |
| 4 | 人物行为三重驱动（经历+利益+性格） | 角色降智 |
| 5 | 五感细节描写 | 空洞叙述 |
| 6 | 章末留钩子 | 章节不吸引人 |
| 7 | 对话占比 30-50% | 纯叙述枯燥 |
| 8 | "突然""竟然"不超过2次 | AI偷懒转折 |
| 9 | 禁止机械降神 | 不合理转折 |
| 10 | 避免 AI 味套话 | 模板化表达 |

### 4.2 内嵌章前规划

每次写作前，Writer 先生成一个 100-200 字的简短规划：

```
本章目标: [1句话]
关键场景: [2-3个]
情绪弧线: [从X到Y]
需回收伏笔: [哪些]
需埋设伏笔: [哪些]
```

这个规划不输出给用户，只作为后续正文生成的引导。

### 4.3 输出约定

- 只输出章节正文，不要标题/前言/后记/注释
- 中文引号「」代替英文引号
- 对话独立成段

## 五、StateManager (3c) — 状态管理

### 写后流程

```python
StateManager.post_write(chapter_num, chapter_text):
  1. 保存 → projects/书名/chapters/chapter_0001.md
  2. 提取摘要 → 追加到 chapter_summaries.md
  3. 更新写状态 → chapter_number += 1
  4. 返回保存路径
```

摘要格式：
```
1. 沈安在破庙醒来发现记忆被替换，决定查明真相离开破庙。
```

每行一个章节，便于 ContextBuilder 取最近 5 行。

## 六、CLI 设计

```bash
# 写第 5 章
python main.py write --name "无名密探" --chapter 5

# 写第 5 章 + 带指导
python main.py write --name "无名密探" --chapter 5 --guidance "本章揭示阿沅的真实身份"

# 指定字数
python main.py write --name "无名密探" --chapter 5 --words 4000

# 批量 (Phase 3 不支持，未来扩展)
python main.py write --name "无名密探" --from 3 --to 10
```
