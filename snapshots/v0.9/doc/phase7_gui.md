# Phase 7 — GUI（Streamlit Web 界面）

> 版本：0.1（设计稿）
> 日期：2026-05-24
> 状态：设计阶段

---

## 一、设计目标

为 InkEdge 提供一个人类可用的图形界面。CLI 是给 AI 和自动化用的，GUI 是给人用的。根据 AGENTS.md 规约：CLI 和 GUI 功能必须同步，GUI 是纯调度层不写业务逻辑。

---

## 二、页面规划

### 2.1 页面清单

| 页面 | 路由 | 功能 | 对应 CLI |
|------|------|------|----------|
| 项目仪表盘 | `/` | 已有项目列表 + 快速操作入口 | `status` |
| 新建项目 | `/new` | 建书向导（雪花法6步可视化） | `new` |
| 写稿工作台 | `/write/<name>` | 逐章写作 + 上下文预览 + 实时生成 | `write` |
| 风格管理 | `/style/<name>` | 上传作家片段 + 查看/编辑风格指南 | `style` |
| 向量索引 | `/index/<name>` | 可视化索引状态 + 语义搜索测试 | `index` |
| 校验报告 | `/check/<name>` | Phase 5 校验结果展示 | `check` |
| 角色状态 | `/state/<name>` | Phase 6 角色状态面板 | `state` |

#### 新建项目 `/new`

三步向导：
1. **基本信息**：书名、题材下拉框、主题、章数、字数
2. **写作方法**：unified（inkOS 风格）/ snowflake（雪花法），预览各方法说明
3. **建书确认**：显示将调用的 LLM 配置，点击"开始建书"→ 实时显示生成进度

建书完成后自动跳转到项目仪表盘。

## 三、技术方案

### 3.1 技术选型

**Streamlit** — 最适合此类项目：
- 纯 Python，无需 JavaScript 开发
- 组件库丰富（按钮、输入框、文件上传、进度条、图表）
- 与现有 src/ 模块直接 import 调用
- 热重载开发体验

### 3.2 架构原则

```
studio.py（GUI 调度层）
  │  import streamlit as st
  │  纯界面逻辑：渲染/输入/展示
  │
  ├─→ src/agents/architect.py     建书
  ├─→ src/agents/writer.py        写稿
  ├─→ src/agents/style_analyzer.py  风格
  ├─→ src/retrieval/indexer.py    索引
  └─→ src/state/manager.py        状态
```

GUI 零业务逻辑 —— 所有操作委托给 `src/`。

### 3.3 项目结构

```
studio.py                  # Streamlit 入口
pages/
├── 01_new_project.py      # 新建项目页
├── 02_write.py            # 写稿工作台
├── 03_style.py            # 风格管理
├── 04_index.py            # 向量索引
├── 05_check.py            # 校验报告
└── 06_state.py            # 角色状态
```

### 3.4 启动方式

```bash
# CLI
python main.py studio

# 直接启动
streamlit run studio.py
```
