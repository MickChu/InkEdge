# Phase 3.5 — Style System（风格控制）

> 状态：✅ 已完成（2026-05-23）
> 测试：18/18 通过

---

## 一、设计目标

让 InkEdge 写的文字"像某个作家写的"。不同于 Phase 8 的类型知识注入（解决骨头），Style System 解决的是**腔调**——叙事语气、对话风格、描写密度、节奏控制。

---

## 二、StyleAnalyzer — 六步推导法

### 2.1 分析流程

```
输入: 作家片段（2000-5000字）
  │
  ├─ 第1步  叙事距离识别     # 全知/限知/第一人称？叙述者与故事的距离？
  ├─ 第2步  句式指纹         # 句长分布、断句习惯、排比/对偶频率
  ├─ 第3步  对白风格         # 占比、长短、潜台词密度、方言/口语化程度
  ├─ 第4步  描写策略         # 五感偏好、场景切换速度、留白程度
  ├─ 第5步  节奏控制         # 张弛交替模式、章节收束方式
  └─ 第6步  词汇谱系         # 高频词、禁词、时代感、文言/白话比例
  │
  ▼
输出: style_guide.md
```

### 2.2 输出的 8 个维度

```markdown
# 风格指南: 冷峻精密叙事

## 1. 叙事语气
叙述者冷眼旁观，不替角色解释动机。
句子短硬，少用形容词，多用动词。

## 2. 对话风格
对白占比约 35%，极少超过 5 轮。
潜台词密度高，说半句留半句。

## 3. 场景描写
视触觉优先（铁锈味、冰凉石地），听觉次之。
场景切换快，不恋战。

## 4. 节奏控制
每章至少一个"安静段落"——角色独处/沉思。
高潮段落信息密度突然升高。

## 5. 词汇特征
高频: "冷""硬""沉""暗"
禁用: "美好""温暖""幸福"（属于本风格的违和词）

## 6. 句式特征
短句为主（平均 12-18 字），长句仅用于描写。
极少排比，不对偶。

## 7. 情绪表达
情绪从不直接命名（不写"他很愤怒"）。
用身体反应和动作替代。

## 8. 去AI味对照表
| AI高频词 | 本风格替代 |
|----------|-----------|
| 突然     | 直接用动作起句 |
| 微微一笑 | 具体的面部细节 |
| 值得注意的是 | （本风格不用） |
```

### 2.3 去 AI 味对照表

每个风格指南自动生成一个对照表，把该作家的习惯表达与 AI 高频词对应起来，Writer 在写作时自动避开 AI 味词汇。

## 三、集成

### 3.1 与 Writer 的集成

ContextBuilder 自动检测 `projects/书名/style_guide.md` 是否存在，存在则加载并注入：

```
Writer 系统提示词中的风格段落：
"如果上下文包含「风格指南」部分，你必须严格遵循其中描述的
所有风格特征。风格指南中的「去AI味对照表」和「六步推导法」
内化为写作习惯，不机械套用。"
```

### 3.2 上下文预算

风格指南占 2000 字预算（在 ContextBuilder 的 budgets 中）。

## 四、CLI 设计

```bash
# 从文件分析
python main.py style --name "无名密探" --file "金庸片段.txt"

# 直接粘贴文本
python main.py style --name "无名密探" --text "沈安醒来的时候..."

# 查看当前风格指南
python main.py style --name "无名密探" --show
```

## 六、与 Phase 8 的关系

| | Phase 3.5 Style | Phase 8 类型知识 |
|---|---|---|
| 解决什么 | "读起来像谁"（腔调） | "设定该长什么样"（骨头） |
| 输入 | 作家文本片段 | 类型领域知识模块 |
| 作用端 | Writer 输出端 | Architect 建书端 + Writer 上下文 |
| 关系 | 互补不重叠 | 互补不重叠 |
